documentation of assumptions and trade-offs.

## Part 1: Code Review & Debugging

## Part 2: Database Design
- Designed a scalable schema for a B2B inventory system

- Identified missing requirements and documented assumptions

Focus areas:
- Data integrity
- Relationships and constraints
- Scalability

## Part 3: API Implementation
- Implemented a low-stock alert API for multi-warehouse companies
- Applied business rules such as variable thresholds and recent sales activity

Focus areas:
- API design
- Business logic
- Edge-case handling
